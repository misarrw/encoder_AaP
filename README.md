### Некоторые правила, которые необходимо учитывать при работе с репозиторием
- нельзя пушить свою папку `build`, ее в принципе не должно быть в репозитории
- при каждом пулле себе на локальный репозиторий этого удаленного репозитория переделывайте `build`, дабы избежать ошибок
- коммиты лучше делать поэтапно, не добавляем за один коммит все измененные файлы, а только те, что относятся к конкретной фиче, для остальных делаем новый коммит
- коммиты лучше называть в соответствии с конкретной фичей, так будет удобнее при тестировании и дебаге
